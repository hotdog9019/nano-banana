# Ticket Triage Assistant

Итоговый мини-проект по курсу «Инженерия Искусственного Интеллекта»: сервис для автоматической классификации обращений в поддержку.

## 1. Паспорт проекта

- **Название проекта:** Ticket Triage Assistant
- **Автор:** Элизбарян Артём Гайкович
- **Группа:** ЭФБО-14-24
- **Контакт:** powerfullhotdog@gmail.com

Проект решает задачу первичной маршрутизации обращений пользователей по пяти категориям: `account_access`, `billing`, `technical_issue`, `how_to`, `feature_request`. Используется синтетический русско-английский датасет, сравнение нескольких baseline/ML-подходов, TF-IDF признаки по символьным n-граммам и линейный классификатор `SGDClassifier` с логистической функцией потерь. Результат проекта - воспроизводимый пайплайн обучения и HTTP-сервис с эндпоинтами `/health` и `/predict`.

## 2. Структура проекта

- `data/support_tickets.csv` - демонстрационный датасет обращений.
- `configs/config.yaml` - пути, параметры сплита, модели и сервиса.
- `configs/.env.example` - пример переменных окружения без секретов.
- `src/` - код загрузки данных, обучения, инференса и HTTP-сервиса.
- `tests/` - sanity-тесты данных, обучения и контракта предсказания.
- `artifacts/` - сохраненная модель, метрики и примеры предсказаний после обучения.
- `notebooks/` - место для EDA и экспериментальных материалов.
- `report.md` - отчет по постановке задачи, экспериментам и результатам.
- `self-checklist.md` - самопроверка проекта перед сдачей.

## 3. Установка

Требуется Python `>=3.10`.

```bash
cd project
python -m venv .venv
.venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

На Linux/macOS активация окружения: `source .venv/bin/activate`.

## 4. Запуск

Обучить модель и пересоздать артефакты:

```bash
cd project
python -m src.train
```

После обучения появятся:

- `artifacts/ticket_classifier.joblib`
- `artifacts/metrics.json`
- `artifacts/sample_predictions.csv`

Запустить сервис:

```bash
cd project
python -m src.service
```

По умолчанию сервис поднимается на `http://127.0.0.1:8000`.

Проверка состояния:

```bash
curl http://127.0.0.1:8000/health
```

Пример предсказания:

```bash
curl -X POST http://127.0.0.1:8000/predict ^
  -H "Content-Type: application/json" ^
  -d "{\"text\":\"I was charged twice for the same subscription\"}"
```

Ответ содержит категорию, confidence и распределение вероятностей по классам.

## 5. Данные

Датасет `data/support_tickets.csv` содержит 100 синтетических обращений на русском и английском языках. В нем есть текст обращения, целевая категория и приоритет. Данные не содержат персональных или конфиденциальных сведений и подходят для быстрого воспроизведения пайплайна без внешних скачиваний.

## 6. Тесты

```bash
cd project
pytest tests -q
```

Тесты проверяют наличие обязательных колонок и классов, стратифицированный split, обучение модели и контракт ответа предсказания.

## 7. Демонстрация на защите

1. Показать структуру `project/`: `data`, `configs`, `src`, `tests`, `artifacts`.
2. Запустить `python -m src.train` и показать метрики из `artifacts/metrics.json`.
3. Запустить `python -m src.service`, открыть `/health` и отправить 1-2 запроса в `/predict`.
4. Кратко объяснить, почему финальная модель лучше baseline и где проект ограничен маленьким датасетом.

## 8. Ограничения и дальнейшая работа

Текущая версия использует синтетический датасет, поэтому качество ограничено разнообразием сгенерированных примеров. В дальнейшем проект можно улучшить реальными обезличенными обращениями, кросс-валидацией, расширенным feature engineering, мониторингом качества и FastAPI/Swagger-интерфейсом.

## 9. Текущие результаты

Последний прогон `python -m src.train` сохраняет полное сравнение моделей в `artifacts/metrics.json`. Основные результаты:

| Модель | Accuracy | Macro-F1 |
|---|---:|---:|
| Baseline `most_frequent` | 0.200 | 0.067 |
| TF-IDF word + ComplementNB | 0.600 | 0.591 |
| TF-IDF char + LinearSVC | 0.700 | 0.665 |
| TF-IDF char + SGDClassifier | 0.700 | 0.665 |
